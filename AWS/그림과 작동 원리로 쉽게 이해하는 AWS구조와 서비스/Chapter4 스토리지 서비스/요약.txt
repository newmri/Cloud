Amazon Simple Storage Service(S3)은 AWS에서 제공하는 객체 스토리지 서비스
객체 스토리지는 폴더 구조가 아닌 객체 키로 데이터를 고유하게 식별
버킷: 데이터를 저장하는 장소 (버킷 이름은 전 세계에서 고유 해야함)
객체: 버킷에 저장된 데이터 본체
키: 객체의 저장 URL 경로 s3://버킷이름/키이름/객체이름

스토리지 클래스: 비용과 가용성이 다름

S3의 버전 관리기능을 활성화하면 이력 정보가 보존, 삭제 할경우 삭제 마커가 부여, 버전 ID를 지정해 삭제 가능

S3 접근 방법
1. 사용자 정책: 데이터를 조작하는 쪽을 제어하는 설정 (데이터 검색 가능, 업로드 불가)
2. 접근 제어 목록: 버킷단위로 접근 제어 (데이터 A는 계정 X에 허용)
3. 버킷 설정: 버킷 단위 설정 (특정 IP로 부터 접근 허가)

Amazon Elastic Block Store(EBS)는 EC2와 함께 사용하는 스토리지 서비스 (EC2의 HDD/SDD와 같은 역할)

EBS는 빠르지만 하나의 EC2에만 연결 가능
S3는 많은 클라에서 동시에 사용 가능하지만 API를 이용하므로 느림
이를 해결: Amazon Elastic File System(EFS) 여러 EC2에서 사용 가능

Amazon FSx: 파일 서버 구축
AWS Stroage Gateway: 온프레미스 서버 기기 혹은 가상 서버에 설치해서 온프레미스 기기와 AWS의 S3, FSx, EBS를 직접 연결
AWS Transfer Family: FTP 기반 서버를 위한 서비스
AWS DataSync: 온프레미스와 AWS 혹은 AWS 스토리지 서비스간 데이터 전송을 위한 서비스
AWS Snow Family: 물리 스토리지를 AWS에서 빌려 거기에 데이터를 저장하고 AWS로 반환하면
해당 데이터를 직접 AWS 내의 스토리지에 옮겨 주는 서비스
