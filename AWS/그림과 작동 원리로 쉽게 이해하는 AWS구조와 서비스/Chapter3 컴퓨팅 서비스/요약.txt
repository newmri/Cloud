EC2는 가상 서버를 생성 하는 서비스

EC2를 인터넷에 공개하려면
1. 퍼블릭 서브넷에 배치
2. 퍼블릭 IP 주소를 부여
3. 보안 그룹에서 외부 접근 허가

외부 -> EC2 = 인바운드 규칙
EC2 -> 외부 = 아웃바운드 규칙

0.0.0.0/0은 모든 네트워크
10.0.0.0/16은 내부 네트워크 (사설 네트워크)

EC2 Auto Scaling으로 CPU 사용률에 맞춰 서버 추가 가능

AWS Lambda: 서버리스 컴퓨팅 서비스, 인프라는 AWS에서 관리 사용자는 코드 개발에만 집중
API Gateway와 결합해 간단한 웹 프로그램 제작 가능
사용자 -> HTTP 요청 -> API Gateway -> JSON 데이터를 Lambda 함수에 전달
S3 버킷에 데이터가 저장되면 Lambda로 자동 처리 가능
스케줄은 EventBridge를 통해 가능 Ex) 매일 12시에 Lambda 함수 실행
Lambda는 S3에 데이터를 저장하거나 EC2 시작/중지 등 목적으로 사용되는 경우가 많지만 IAM역할 형식으로 권한 부여 필요

가상화는 아파트와 같지만 컨테이너는 쉐어 하우스와 같음 OS와 물리자원은 컨테이너 별로 공유
컨테이너는 컨테이너 이미지 실행
컨테이너 오케스트레이션: 여러 컨테이너를 관리
AWS에서 컨테이너 시스템을 구축하는 경우: ECS와 EKS가 있음
ECS: AWS의 관리형 컨테이너 오케스트레이션
ECS를 이용할 때 플랫폼은 EC2와 Fargate 두가지 선택 가능
Fargate: aws에서 서버를 관리
EKS: 쿠버네티스 사용시 사용해야함, 노드는 EC2와 Fargate 두가지 선택 가능

Pod: 여러 개의 컨테이너가 함께 동작하는 단위
Node: 파드를 실행하는 물리적/가상 머신
클러스터 → 노드 → 파드 → 컨테이너

클러스터(Kubernetes Cluster): 여러 개의 노드(Node)로 구성됨.
노드(Node): 파드를 실행하는 물리적/가상 머신.
파드(Pod): 하나 이상의 컨테이너를 포함하는 논리적 실행 단위.
컨테이너(Container): 실제 애플리케이션이 동작하는 환경.
